# Лабораторная работа 1 — Computer Vision

## Тема
Обнаружение и распознавание объектов с использованием **Ultralytics YOLO11**.

## Что находится в репозитории
Основной файл работы — `lab1.ipynb`.  
Он содержит полный сценарий выполнения лабораторной работы и запускается последовательно **с начала до конца**.

В работе используются следующие директории и файлы:

```text
lab1.ipynb
lab1_cv_yolo/
├── raw/                  # исходные данные
├── prepared/             # датасет после подготовки к формату YOLO
├── runs/                 # результаты обучения и предсказаний
├── results_summary.csv   # сравнение baseline и improved baseline
└── final_comparison.csv  # итоговое сравнение всех подходов
```

Датасет уже расположен в папке:

```text
lab1_cv_yolo/raw/
```

## Как запустить работу
1. Открыть `lab1.ipynb`.
2. Запустить ноутбук **целиком сверху вниз**.
3. Не пропускать ячейки и не запускать блоки выборочно после перезапуска kernel.
4. Дождаться завершения всех этапов.

## Что делает ноутбук

### 1. Подготавливает рабочие директории
В начале создаются или используются рабочие папки:
- `raw/` — исходные данные;
- `prepared/` — подготовленный датасет;
- `runs/` — результаты обучения.

### 2. Распаковывает и анализирует датасет
Ноутбук берет архив из папки `raw/`, распаковывает его и просматривает структуру файлов.  
На этом этапе определяется:
- где лежат изображения;
- где лежит разметка;
- какие классы присутствуют в датасете.

### 3. Приводит данные к формату YOLO
После анализа структуры ноутбук:
- собирает пары `изображение -> разметка`;
- формирует train/val split;
- создает `data.yaml`;
- сохраняет готовый датасет в формате YOLO в папку `prepared/`.

### 4. Выполняет визуальную проверку разметки
Показываются несколько изображений с bounding boxes, чтобы убедиться, что разметка была обработана корректно.

### 5. Обучает baseline-модель
Далее обучается baseline-модель **YOLO11n**, после чего выполняется валидация и рассчитываются основные метрики качества:
- `Precision`
- `Recall`
- `mAP@0.5`
- `mAP@0.5:0.95`

### 6. Обучает улучшенную модель
После baseline запускается улучшенная конфигурация на **YOLO11s** с более сильной моделью и расширенными настройками обучения.  
Затем снова рассчитываются метрики качества и результаты сравниваются с baseline.

### 7. Формирует таблицу сравнения baseline и improved baseline
Результаты двух моделей сохраняются в файл:

```text
lab1_cv_yolo/results_summary.csv
```

### 8. Выполняет предсказания на валидационных изображениях
Ноутбук запускает лучшую модель на нескольких изображениях из validation-выборки и показывает результат детекции.

### 9. Запускает самостоятельную обучаемую модель
Дополнительно реализована простая самостоятельная обучаемая модель, которая:
- извлекает признаки объектов;
- обучается на train-части датасета;
- оценивается на validation-части;
- сравнивается с моделями YOLO.

### 10. Формирует итоговую таблицу сравнения
В конце ноутбука создается итоговая таблица, где сравниваются:
- самостоятельная обучаемая модель;
- `YOLO11n baseline`;
- `YOLO11s improved`.

Итоговая таблица сохраняется в файл:

```text
lab1_cv_yolo/final_comparison.csv
```

## Что должно получиться после запуска
После полного выполнения ноутбука должны быть получены:
- подготовленный датасет в `lab1_cv_yolo/prepared/`;
- результаты обучения и предсказаний в `lab1_cv_yolo/runs/`;
- файл `lab1_cv_yolo/results_summary.csv`;
- файл `lab1_cv_yolo/final_comparison.csv`.

## Важное замечание
Ноутбук нужно запускать именно **последовательно с начала**, так как отдельные ячейки используют переменные и пути, созданные на предыдущих этапах.

## Краткий итог
`lab1.ipynb` реализует полный воспроизводимый пайплайн лабораторной работы:
- подготовка датасета;
- обучение baseline;
- обучение улучшенной модели;
- запуск собственной обучаемой модели;
- расчет метрик;
- сравнение результатов;
- сохранение итоговых артефактов.
